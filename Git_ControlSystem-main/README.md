# Git_ControlSystem

A practical demonstration of Git version control basics including initialization, staging, committing, and pushing to GitHub.

## Project Description

This repository contains a simple hello_world program used to practice fundamental Git commands and workflows. It demonstrates the complete Git workflow from local repository setup to pushing code to a remote GitHub repository.

## Files

- `hello_world.py` - Simple Python script demonstrating basic output
- `README.md` - Project documentation
- `example-repo.txt` - Repository URL reference

## Installation

```bash
git clone https://github.com/Qhayiya29-coder/Git_ControlSystem.git
cd Git_ControlSystem
```

## Usage

To run the hello_world program:

```bash
python hello_world.py
```

Expected output:
```
Git is awesome!
```

## Git Commands Used

This project demonstrates the following Git commands:

- `git init` - Initialize a new Git repository
- `git add` - Stage changes for commit
- `git commit` - Save snapshots of changes with meaningful messages
- `git status` - Check the status of working directory and staging area
- `git log` - View commit history
- `git remote add` - Connect local repository to remote GitHub repository
- `git push` - Upload local commits to remote repository

## Learning Objectives

This project demonstrates:

1. **Local Repository Setup**: Creating and initializing a Git repository
2. **File Tracking**: Adding files to Git's staging area
3. **Version Control**: Making changes and committing them with descriptive messages
4. **Remote Integration**: Connecting local repository to GitHub
5. **Collaboration**: Pushing code to remote repository for sharing and collaboration
